# dict 和 set

## `dict` 的实现及其导致的结果

### 键必须是可散列的
1. 支持`hash()`函数，并且通过 `__hash__()` 方法所得到的散列值是不变的。
2. 支持通过 `__eq__()` 方法来检测相等性。
3. 若 `a == b` 为真, 则 `hash(a) == hash(b)` 为真.

### 字典在内存上的开销巨大

### 键的查询很快

### 键的次序取决于添加顺序
如果在`key1`和`key2`被添加到字典里的过程中有冲突发生的话，这两个键出现在字典里的顺序是不一样的。

### 往字典里添加新键可能会改变已有键的顺序

## `set` 的实现及其导致的结果

* 集合里的元素必须是可散列的
* 集合很消耗内存
* 可以很高效的判断元素是否存在于某个集合。
* 元素的次序取决于被添加到集合中的次序
* 往集合中添加元素，可能会改变集合里已有元素的次序。
